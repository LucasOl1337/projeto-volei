Licao 0002

Dashboard nao e gaveta

Nesta licao voce aprende uma decisao importante de produto: o Dashboard deve mostrar poucos indicadores principais. Ele nao deve virar uma tela com tudo que o app sabe fazer.

Produto

Uma pergunta por tela

O Dashboard responde: como estou agora? Por isso ele ficou com Fisico, Tecnico e Fundamento.

Codigo

Dados antes de layout

Primeiro criamos uma lista de indicadores. Depois renderizamos os cards com map. Isso evita copiar tres blocos quase iguais.

Volei

Alerta antes da causa

Se Fundamento esta baixo, o Dashboard mostra o alerta. A causa fica em Exercicios, Indicadores ou Relatorios.

O raciocinio da mudanca

1

Remover ruido

Treinos recentes, relatorios e acoes sairam do Dashboard porque exigem leitura detalhada.

2

Escolher tres sinais

Fisico, Tecnico e Fundamento viraram os sinais principais para abrir o app e entender o estado geral.

3

Mandar detalhes para telas proprias

Biblioteca de exercicios, indicadores e relatorios explicam o que fazer depois que um numero chama atencao.

Exercicio rapido

Responda antes de abrir a resposta.

Se o indicador Fundamento esta baixo, o que o Dashboard deve fazer?

Mostrar o alerta principal. A investigacao deve acontecer em outra tela, como Exercicios, Indicadores ou Relatorios.

Para ler depois

Para entender o padrao de renderizar uma lista de cards a partir de dados, leia a documentacao oficial do React sobre rendering lists. Para conectar isso ao nosso projeto, revise tambem a referencia rapida de Dashboard com indicadores.

Pergunte ao agente se quiser transformar esse mesmo raciocinio em codigo passo a passo.
